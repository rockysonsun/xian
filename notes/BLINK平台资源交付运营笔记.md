## 一、项目类型总览

| 项目类型 | 资源类型 | 硬件要求 | 典型场景 |
|---------|---------|---------|---------|
| **中小节点** | 移动端/小型设备 | ARM/X86低配置 | 手机、平板、机顶盒、路由器 |
| **大节点** | 汇聚/专线服务器 | X86高配置+大带宽 | 数据中心、机房专线 |
| **X项目** | 云主机资源 | X86≥12C/32G/SSD≥1TB | 云服务器部署 |
| **Y项目** | 物理机资源 | X86≥12C/32G/SSD≥1TB | 自有物理服务器 |

## 二、平台核心功能

### 2.1 资源上报
- **资源信息**: 查看需求项目、资源类型、招募要求、需求量级
- **申请记录**: 已申请项目列表及上报结果

### 2.2 设备管理
- **接入设备**: 自动检测节点服务、查看运行状态
- **设备资产**: 设备交付管理、状态监控、报障处理
- **问题设备**: 异常汇总、校验失败处理（下线/更新/更换/恢复）
- **设备分组**: 标签化自定义分组

### 2.3 下载包管理
- 手动构建B站资源需求系统安装包
- 支持多架构(X86/ARM32/ARM64/MIPS)部署包下载

## 三、部署方式对比

### 3.1 中小节点部署模式

| 模式 | 运行环境 | 权限要求 | 适用设备 | 依赖 |
|-----|---------|---------|---------|-----|
| **SHELL** | 宿主机直接运行 | root | Linux服务器 | iptables、docker |
| **DOCKER** | 隔离容器 | root | 支持docker的设备 | docker |
| **PROOT** | 非root容器 | 非root | ARM设备 | 业务rootfs(支持iptables) |
| **CHROOT** | root容器 | root | ARM设备 | BLINK自带rootfs |
| **Android** | 纯安卓环境 | root/非root | 安卓9+设备 | NDK编译版本 |

### 3.2 大节点/X/Y项目部署方式

| 方式 | 步骤 | 说明 |
|-----|-----|-----|
| **系统镜像** | 安装镜像→自动启动blink→自动拉取业务 | 推荐方式 |
| **部署脚本** | CentOS7可直接运行脚本部署 | 免镜像安装 |

## 四、关键配置参数

### 4.1 环境变量（通用）

```bash
# ===== 必填(部署) =====
BLINK_DEPLOY_PATH="/data/blink_agent"        # blink安装路径
BLINK_DOCKER_MOUNT_POINT="/data/blink_data"  # 业务容器映射目录

# ===== 必填(资产信息) =====
BLINK_VENDOR_DEVICE_ID="ABCD1234"   # 产商设备ID(唯一标识)
BLINK_NUMBER_OF_ETH=1               # 网口线路数量
BLINK_SINGLE_BANDWIDTH=1000         # 单线路带宽(M)
BLINK_PROVINCE="Shanghai"           # 省份(英文)
BLINK_ISP="CTCC"                    # 运营商(英文)
BLINK_DEVICE_TYPE="PBOX"            # 设备类型(英文)
BLINK_SCHED_SCOPE="province"        # 调度方式(province/crossisp/缺省跨省)

# ===== 可选(多线路支持) =====
BLINK_ETH_LIST="ppp0:ppp1:ppp2"     # 多网卡列表
BLINK_DOCKER_MOUNT_LIMIT="64:64:64" # 缓存大小限制(G)
BLINK_COVER_ISP="100,100,100,0,100,100"  # 精准调度比例(省内cm,cu,ct/省外cm,cu,ct)
BLINK_BANDWIDTH_COLLECT_TYPE="nf_conntrack"  # 带宽采集方式
```

### 4.2 资产信息文件格式

**文件路径**:
- Docker容器: `/storage/asset`
- 物理设备: `${BLINK_DEPLOY_PATH}/asset`

**格式**: KEY=VALUE (每行一个，无空格)

```
VENDOR_DEVICE_ID=ABCD1234
NUMBER_OF_ETH=1
SINGLE_BANDWIDTH=100
PROVINCE=Shanghai
ISP=CTCC
DEVICE_TYPE=PBOX
SCHED_SCOPE=province
```

## 五、命名规范

### 5.1 大节点主机命名
```
mcdn-{渠道缩写}-{省份城市缩写}-{运营商缩写}-{业务标识}-{后缀}

# 示例: bili渠道、江苏苏州、电信、mp2sp业务
mcdn-bili-jssz-dx-mp2sp-001
```

### 5.2 X/Y项目主机命名
```
neep-{供应商}-{省份}-{实际运营商}-{业务标识}-{编号后缀}

# 示例: jkwl供应商、cq省份、cmcc运营商、x业务
neep-jkwl-cq-cmcc-x-350
```

### 5.3 缩写规范

| 类型 | 缩写 | 说明 |
|-----|-----|-----|
| **运营商** | dx | 电信 |
| | cu | 联通 |
| | cmcc | 移动 |
| **业务标识** | v | 专线业务(单线路≥3G) |
| | mp2sp | 汇聚公网资源 |
| | mnet | 汇聚内网资源 |
| | x/y | XY项目标识 |

## 七、硬件要求对比

| 配置项 | 中小节点 | 大节点 | X项目(云主机) | Y项目(物理机) |
|-------|---------|-------|-------------|-------------|
| **架构** | ARM32/ARM64/X86/MIPS | X86 | X86 | X86 |
| **CPU** | 依业务而定 | 高性能 | ≥12核 | ≥12核 |

| **NAT要求** | 依业务而定 | - | 全锥及以上 | 全锥及以上 |
| **Docker** | 部分模式需要 | 是 | 必须 | 必须 |
| **root权限** | 部分模式需要 | 需要 | 需要 | 需要 |

格式: `省内cm,省内cu,省内ct,省外cm,省外cu,省外ct`

## 九、常见问题排查

### 9.1 中小节点零跑量/低跑量
1. 排查业务进程是否启动
2. 排查设备网络是否正常
3. 查看 `centaurs.json` 文件信息
4. 检查NAT类型(对称型设备占比需<5%)
5. 检查重传率(应<5%)、CPU/内存/负载
6. 检查TCP重传(<3%)、UDP丢包(<3%)

### 9.2 大节点资产未绑定
1. 确认网络: `telnet 218.94.210.101 51279`
2. 确认blink agent: `systemctl status blink`
3. 确认主机名与平台提交信息一致

### 9.3 业务部署失败
1. 检查磁盘挂载: `lsblk`
2. 检查磁盘格式化是否过慢
3. 检查镜像拉取是否超时

### 9.4 自动下线原因
| 异常 | 阈值 | 排查命令 |
|-----|-----|---------|
| CPU负载 | >80% | `top`、`lscpu` |
| 网络重传率 | >10% | - |
| 网络丢包率 | >10% | `ping -I ethX <目标>` |
| 网络线路可用率 | <50% | `telnet IP 端口` |
| 磁盘故障率 | >50% | `dmesg -T`、`iostat` |
| 大节点错误率 | 后5%且错误率>均值且>1000 | - |
| 心跳异常 | >3分钟无心跳 | `supervisorctl restart portal-pin` |

## 十一、异网改造流程

```bash
# 1. 进入blink部署目录
cd ${BLINK_DEPLOY_PATH}

# 2. 修改环境变量
export blink_cover_isp="100,100,100,0,100,100"

# 3. 修改配置文件
vim plugins/blink_default/config
# 修改 COVER_ISP=xxx

# 4. 重启业务进程
plugins/blink_default/control restart
```
